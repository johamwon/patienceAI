# 患癌知光 — 修订版实施计划（Implementation Plan v2）

> **状态**：待批准  
> **周期**：6 周（约 1.5 个月）  
> **团队**：3–5 人  
> **目标**：大赛可演示的 MVP  
> **重大变更**：使用 KnowS AI 结构化医学证据检索 API 替代自建检索层，聚焦多智能体通俗化引擎

---

## 执行摘要

```
阶段 0  准备（3 天）    — API Key 注册、环境搭建、技术栈确认
阶段 1  检索集成（1 周）  — KnowS AI 6 类医学证据源接入 + 查询重写 + 前端基础
阶段 2  通俗化引擎（2 周）— 五位一体 Simplification Loop（核心创新）
阶段 3  安全合规（1 周）  — 幻觉防御 + 风险路由 + 审计日志
阶段 4  评测演示（1 周）  — 离线评测 + Demo 准备 + 文档编写
```

**关键变更说明：**

| 原方案 | 修订方案 | 原因 |
|---|---|---|
| 自建 12 个 MCP 工具 + BM25/MedCPT 检索管线 | **直接使用 KnowS AI API**（6 类权威医学源：论文/会议/指南/临床试验/药品说明书） | 大赛提供现成 Skill，避免重复造轮子，节省 3–4 周 |
| 10 周完成 | **6 周完成** | 检索层工作量外包给 KnowS AI，聚焦差异化能力 |
| 4–6 人团队 | **3–5 人团队** | 检索层无需专人维护 |
| 自建 UMLS 词典 | **KnowS AI 返回结构化实体 + 轻量 UMLS 对齐** | MVP 阶段简化，优先保证核心创新点 |

| 医学证据类型 | KnowS AI 端点 | 调用时机 | 返回内容 |

| 英文文献 | `POST /evidences/ai_search_paper_en` | 用户查询涉及国际研究 | 最多 40 条结构化证据（标题/作者/PMID/期刊/摘要/来源类型） |

**KnowS AI 调用示例：**

```python
import os
import requests

KNOWS_BASE_URL = "https://api.nullht.com/v1"
KNOWS_API_KEY = os.environ.get("KNOWS_API_KEY")  # 可选

headers = {"Content-Type": "application/json"}
if KNOWS_API_KEY:
    headers["Authorization"] = f"Bearer {KNOWS_API_KEY}"

# 检索英文论文
resp = requests.post(
    f"{KNOWS_BASE_URL}/evidences/ai_search_paper_en",
    headers=headers,
    json={"query": "NPM1 mutation therapeutic predictor MDS"},
)
evidence = resp.json()  # 结构化证据列表
```

## 阶段 2：通俗化引擎（Week 2–3）★ 核心创新

**目标**：实现五位一体多智能体 Simplification Loop，这是产品的核心差异化

### Sprint 2-A：Simplification Loop 核心（Week 2）

| 任务 | 功能 | 核心逻辑 | 验收标准 |
|---|---|---|---|
| 2-A-01 | Layperson Agent | 扫描医学证据文本，识别生涩术语与长难句，生成注意力掩码 M | 对"5 年 OS 率 45.2%（95% CI: 38.1-52.3, p<0.001）"正确标注"OS 率""95% CI""p 值" |
| 2-A-02 | Medical Expert Agent | 接收掩码 M，调用 UMLS 精简子集 + 医学知识库，为生涩术语匹配权威释义 | "OS" → "总生存期，指从治疗开始到死亡的时间" |
| 2-A-03 | Simplifier Agent | 交叉注意力缝合掩码 M + 专家释义 + 原始文本，用主动语态短句重塑 | 输出草稿：核心概念不变，句式大幅简化 |
| 2-A-04 | Language Clarifier Agent | 替换学术书面词和生僻连词 | "呈现出具有显著统计学意义的递减关联性" → "显著降低" |
| 2-A-05 | Redundancy Checker Agent | 计算自注意力熵值，精简啰嗦表达；计算编辑距离 EditDistance(T_i, T_{i-1}) | 相邻迭代编辑距离 < ε = 0.05 时退出循环 |
| 2-A-06 | Simplification Loop 编排器 | 串联 5 个智能体，管理收敛控制与最大迭代次数（≤ 5 轮） | 完整循环跑通，输出收敛文本 |

### Sprint 2-B：可读性质量门控（Week 2–3）

| 任务 | 功能 | 验收标准 |
|---|---|---|
| 2-B-01 | ARI / FKGL / SMOG 计算模块 | 正确计算三大可读性指标 |
| 2-B-02 | 可读性质量门控 | 集成到 Simplification Loop 末尾，FKGL > 10 时触发重新润色 | 最终输出 FKGL ≤ 10 |
| 2-B-03 | 事实一致性检查 | 验证通俗化文本是否保留了原始证据的核心事实 | 核心事实保留率 ≥ 0.92 |

### Sprint 2-C：三层结构化输出（Week 3）

| 任务 | 功能 | 核心逻辑 | 验收标准 |
|---|---|---|---|
| 2-C-01 | 一句话结论生成 | Prompt 约束：≤ 50 字，只说患者最关心的要点 | "免疫治疗联合化疗已成为晚期肺腺癌标准一线方案之一，约 45% 患者获益" |
| 2-C-02 | 证据卡片生成 | 结构化输出：研究类型 / 样本 / 干预 / 结局 / 局限性 / PMID | 每条证据 100% 绑定溯源链接 |
| 2-C-03 | 患者通俗解释 | 模板：是什么 → 证据说明什么 → 对你意味着什么 → 何时就医 | 使用类比和日常语言 |
| 2-C-04 | 风险提示注入 | 中风险加黄色提示条，高风险加红色警示条 | 100% 覆盖 |
| 2-C-05 | 免责声明 | 每条回答底部固定显示 | "本内容为医学文献通俗化解释，仅供参考，不构成诊疗建议" |

### 2. API 调用方式

```python
import os
import requests

KNOWS_BASE_URL = "https://api.nullht.com/v1"
headers = {"Content-Type": "application/json"}

# 匿名调用（限流 3 req/s）
# 专属 Key 调用（限流 10 req/s）
if KNOWS_API_KEY := os.environ.get("KNOWS_API_KEY"):
    headers["Authorization"] = f"Bearer {KNOWS_API_KEY}"

def search_evidence(query: str, sources: list[str]) -> list[dict]:
    """并行调用多个 KnowS AI 端点"""
    results = []
    endpoints = {
        "paper_en": "/evidences/ai_search_paper_en",
        "paper_cn": "/evidences/ai_search_paper_cn",
        "guide": "/evidences/ai_search_guide",
        "trial": "/evidences/ai_search_trial",
        "meeting": "/evidences/ai_search_meeting",
        "package_insert": "/evidences/ai_search_package_insert",
    }
    
    for source in sources:
        resp = requests.post(
            f"{KNOWS_BASE_URL}{endpoints[source]}",
            headers=headers,
            json={"query": query},
        )
        results.extend(resp.json().get("evidences", []))
    
    return deduplicate(results)
```

### 3. 为什么选择 KnowS AI 而不是自建检索

| 维度 | 自建检索（原方案） | KnowS AI（修订方案） |
|---|---|---|
| **开发周期** | 3–4 周（12 个 MCP 工具 + 索引构建） | **2–3 天**（直接调用 API） |
| **数据质量** | 依赖公开 API，结构化程度低 | **专为医学证据设计，返回结构化数据** |
| **维护成本** | 需持续维护索引、处理限流 | **大赛提供，无需维护** |
| **大赛契合度** | 通用检索，非大赛核心 | **大赛技术支持方，契合度高** |
| **差异化** | 检索层同质化严重 | **聚焦多智能体通俗化引擎，差异化明显** |
