# 患癌知光 — 分步实施计划（Implementation Plan）

> **状态**：待批准  
> **周期**：10 周（约 2.5 个月）  
> **团队**：4–6 人  
> **目标**：大赛可演示的 MVP

---

## 执行摘要

```
阶段 0  准备周     — 环境搭建、技术选型确认、API 注册
阶段 1  检索基础   — MCP 工具集 + BM25/向量检索 + 基础前端
阶段 2  智能处理   — 查询重写 + 证据分级 + 三层结构化输出
阶段 3  通俗化引擎 — 五位一体 Simplification Loop + 可读性检验
阶段 4  安全合规   — 幻觉防御 + 风险路由 + 审计日志
阶段 5  评测演示   — 离线评测 + 在线演示 + 文档编写
```

## 阶段 0：准备周（Week 0）

**目标**：完成所有前置条件，为阶段 1 铺平道路

### T-01：项目初始化

| 任务 | 交付物 | 验收标准 | 预估工时 | 依赖 |
|---|---|---|---|---|
| 0-01 初始化仓库与工程结构 | monorepo 工程（backend + frontend + mcp-tools + eval） | git 仓库可 clone，CI 通过 | 1 天 | 无 |
| 0-02 确定 LLM 基座模型 | `LLM_CONFIG.md`（模型名称、API key 管理方式、fallback 策略） | 至少确认 1 个可用的模型 API（如 OpenAI / 国产替代） | 0.5 天 | 无 |
| 0-03 注册数据源 API | `API_KEYS.md`（NCBI E-utilities、Europe PMC、Semantic Scholar、MedlinePlus） | 所有 API 均通过 smoke test，能正常返回数据 | 1 天 | 无 |
| 0-04 搭建基础 Docker 环境 | `docker-compose.yml`（OpenSearch / pgvector / 后端服务 / 前端） | `docker compose up` 后所有服务健康 | 1 天 | 0-01 |

### T-02：技术栈确认

| 决策项 | 推荐选择 | 备选 | 决策依据 |
|---|---|---|---|
| 后端框架 | Python FastAPI | Node.js / Go | 医疗 NLP 生态丰富（sentence-transformers / QuickUMLS / MedCPT） |
| 检索引擎 | OpenSearch（含向量搜索） | Elasticsearch + pgvector | 统一关键词+向量检索，减少运维复杂度 |
| 向量模型 | BGE-M3 | mGTE | 多语言 dense+sparse+multi-vector 一体化 |
| 医学语义重排 | MedCPT Cross-encoder | BGE Reranker | 医疗领域 SOTA，零样本能力强 |
| LLM 编排 | LangGraph / 自研 orchestration | 纯 LangChain | 多智能体工作流需要显式状态管理 |
| 前端 | React + TypeScript | Vue / Svelte | 团队熟悉度高，生态丰富 |
| 评测 | 自建数据集 + 人工抽检 | 纯自动评测 | 医疗场景必须人工兜底 |

### T-03：数据源验证

| 数据源 | Smoke Test 验证内容 | 通过标准 |
|---|---|---|
| NCBI E-utilities | `esearch` + `efetch` 检索并获取摘要 | 返回 ≥ 3 条结果，含 PMID/标题/摘要 |
| Europe PMC | REST API 检索 + `annotationsByArticleIds` | 返回实体标注 |
| Semantic Scholar | Graph API 检索 | 返回 TL;DR + 引用数 |
| MedlinePlus | Web service 查询 | 返回 easy-to-read 患者材料 |
| WHO/NICE | Web 抓取或 API | 返回指南标题 + 发布日期 |

**阶段 0 完成标志**：所有 API smoke test 通过，Docker 环境启动成功，技术选型锁定

## 阶段 2：智能处理层（Week 3–4）

**目标**：让系统"理解"患者问题，并按证据质量组织回答

### Sprint 2-A：查询重写与意图识别（Week 3）

| 任务 | 功能 | 核心逻辑 | 验收标准 |
|---|---|---|---|
| 2-A-01 | QuickUMLS 实体提取 | 从口语化查询中提取候选医学实体 | 输入"我爸查出肺上有个结节" → 提取"肺结节"实体 |
| 2-A-02 | Levenshtein 模糊匹配 | 候选实体与 UMLS 词典的模糊匹配（阈值 θ） | "肺结节" → 正确映射到 UMLS CUI |
| 2-A-03 | 嵌入余弦相似度对齐 | sentence-transformers 计算查询向量与候选 CUI 向量相似度 | 口语化医学术语到 CUI 映射准确率 ≥ 0.80（冷启动） |
| 2-A-04 | 意图分类器 | 将查询分为：疾病理解 / 治疗进展 / 药物信息 / 检查解释 / 临床试验 / 谣言核验 / 高风险 | 高风险意图识别准确率 ≥ 0.90 |
| 2-A-05 | 风险等级判定 | 基于关键词规则 + 意图分类结果，输出低/中/高/禁止四级 | 高风险问题 100% 被正确标记 |
| 2-A-06 | 查询规范化模块 | 整合上述步骤，输出结构化查询对象 | 端到端：输入口语化查询 → 输出结构化意图，延迟 < 500ms |

**依赖**：阶段 1 的 `umls_align` MCP 工具

### Sprint 2-B：证据分级与质量控制（Week 3–4）

| 任务 | 功能 | 验收标准 |
|---|---|---|
| 2-B-01 | 来源类型标注器 | 对检索结果自动标注 10 类来源类型，准确率 ≥ 0.90 |
| 2-B-02 | 去重引擎 | 强主键（PMID/DOI/PMCID）+ 弱去重（标题指纹/时间窗口/语义相似度），去重率 ≥ 95% |
| 2-B-03 | GRADE 证据分级 | 基于研究类型 + 偏倚检测自动输出高/中/低/极低确定性，与人工标注一致率 ≥ 0.75 |
| 2-B-04 | 偏倚检测规则引擎 | 系统综述 → PRISMA 2020 特征检查；RCT → RoB 2 域式检查 | 规则覆盖 80% 常见偏倚场景 |
| 2-B-05 | 时效性过滤器 | 过旧文献（>10 年，除非里程碑研究）自动降权；撤稿文献标记 | 降权逻辑正确 |

**依赖**：2-A-01 完成

### Sprint 2-C：三层结构化输出模板（Week 4）

| 任务 | 功能 | 验收标准 |
|---|---|---|
| 2-C-01 | 一句话结论生成模板 | Prompt 模板 + 输出约束，确保结论 ≤ 50 字，只说患者最关心的要点 |
| 2-C-02 | 证据卡片生成模板 | 结构化输出：研究类型 / 样本 / 干预 / 结局 / 局限性 / PMID 链接 |
| 2-C-03 | 患者通俗解释模板 | Prompt 模板：是什么 → 证据说明什么 → 对你意味着什么 → 何时就医 |
| 2-C-04 | 引用组装器 | 每个关键句绑定 PMID/DOI，格式化为可点击链接 | 假引用率 < 2% |
| 2-C-05 | 三层回答组装器 | 整合三层输出 + 风险提示 + 免责声明 | 完整示例输出通过评审 |

**依赖**：2-B 完成，LLM 基座模型可用

## 阶段 3：通俗化引擎（Week 5–6）

**目标**：实现核心创新——五位一体多智能体通俗化翻译闭环

### Sprint 3-A：多智能体 Simplification Loop（Week 5）

| 任务 | 功能 | 核心逻辑 | 验收标准 |
|---|---|---|---|
| 3-A-01 | Layperson Agent | 扫描原始医学文本，识别生涩术语与长难句，生成二进制注意力掩码 | 对输入"5 年 OS 率为 45.2%（95% CI: 38.1-52.3, p<0.001）"正确标注出"OS 率""95% CI""p 值"为生涩区间 |
| 3-A-02 | Medical Expert Agent | 接收注意力掩码，调取 UMLS + 知识图谱，为每个生涩术语匹配权威口语化释义 | "OS" → "总生存期，指从治疗开始到死亡的时间" |
| 3-A-03 | Simplifier Agent | 交叉注意力缝合掩码 + 专家释义 + 原始段落，用主动语态短句重塑 | 输出草稿：用通俗语言重写整段，核心概念不变 |
| 3-A-04 | Language Clarifier Agent | 替换学术书面词和生僻连词 | "呈现出具有显著统计学意义的递减关联性" → "显著降低" |
| 3-A-05 | Redundancy Checker Agent | 计算自注意力熵值，精简啰嗦表达；计算编辑距离判断收敛 | 相邻迭代编辑距离 < 阈值 ε = 0.05 时退出循环 |
| 3-A-06 | Simplification Loop 编排器 | 串联 5 个智能体，管理收敛控制与最大迭代次数（≤ 5 轮） | 完整循环跑通，输出收敛文本 |

**依赖**：阶段 2 的三层回答模板 + UMLS 实体对齐能力

### Sprint 3-B：可读性检验与质量门控（Week 5–6）

以下决策点影响实施路径，请在批准前确认：

| 编号 | 决策项 | 选项 A | 选项 B | 推荐 |
