## 一、产品概述

### 1.1 产品愿景

> **让每一位面对疑难杂症的患者，都能读懂属于自己的那束科研之光。**

当患者拿到"癌症""罕见病""疑难杂症"的诊断时，最本能的反应是上网搜索。然而，他们面对的是 4000 万篇 PubMed 英文文献、密集的学术术语、相互矛盾的预印本和充斥噪声的社交媒体。患者健康素养的不均衡与医学文献的高度专业性之间，存在深固的信息不对称——极易产生认知偏差甚至医疗焦虑。

"患癌知光"是一个**检索优先、证据优先、溯源优先、安全优先**的 AI Agent。它不替代医生做诊断，而是把全球权威医学文献中的最新科研动态，翻译成患者能看懂、能核验、能行动的通俗解释。

### 1.2 产品定位

| 维度 | 定位说明 |
|---|---|
| **是什么** | 循证医学文献检索与通俗化解释的 AI Agent |
| **不是什么** | 不是自动诊断系统、不是自动开方系统、不替代执业医师 |
| **法律边界** | Clinical-Ops AI（临床运营与科普教育），明确豁免于 SaMD 注册 |
| **大赛类别** | Agent（多智能体协同）+ MCP（医学文献检索工具协议）+ Skill（可复用通俗化技能包） |

### 1.3 核心价值主张

```

│   找得到  ——  全球权威多源文献，2 秒内精准召回               │

│   敢上线  ——  四层幻觉防御 + 三级风险路由 + 全链路合规审计    │

## 二、背景与问题分析

### 2.1 行业痛点

| 痛点编号 | 问题描述 | 数据支撑 |
|---|---|---|
| P-01 | **词汇孤岛**：患者使用模糊口语化自然语言查询，而权威文献采用高度标准化术语体系，LLM 直接处理时存在语义漂移 | 大语言模型在专门领域复杂多义性表达上表现出对专业上下文理解不足 |
| P-02 | **信息过载**：PubMed 已包含 4000 万+条引文，患者无法自行筛选高质量证据 | 同一研究在预印本、期刊、新闻中重复传播，患者误以为"很多研究都这么说" |
| P-03 | **阅读壁垒**：医学权威出版物处于大学本科或更高阅读层级，而面向患者的解释应控制在高中及以下 | 自动可读性指数（ARI）、FKGL、SMOG 三大量化指标普遍超标 |
| P-04 | **幻觉风险**：LLM 生成不符合循证标准的文本可能导致误诊（一致性幻觉）和漏诊（完整性幻觉） | 标准 RAG 系统中，信息冗余或概念交叉可能导致不支持声称发生率上升 |
| P-05 | **合规红线**：一旦系统输出个体化诊断或自动开方，监管属性显著变化 | 中国《互联网诊疗监管细则》明确禁止 AI 替代医师提供诊疗服务 |

### 2.2 目标市场背景

- **WHO** 将健康素养定义为个体"获取、理解、评估并使用信息和服务"的能力，并强调信息提供者应以"可理解、可采取行动"的形式提供可信信息
- **中国** 2023 年居民健康素养水平为 29.70%，城乡区域差距显著；国家卫健委已将"科学权威、通俗易懂"的健康科普供给列入 2024—2027 年全民健康素养提升行动重点任务
- **需求场景**：罕见病患者群体（中国预估超 2000 万）、肿瘤新发患者（年新发约 480 万）、疑难杂症就诊群体，对最新科研动态存在迫切需求

## 四、产品目标与成功指标

### 4.1 产品目标

| 目标层 | 目标描述 | 对应大赛评价维度 |
|---|---|---|
| **G1 找得对** | 检索 Recall@10 ≥ 0.90，MRR ≥ 0.85，citation hit rate ≥ 0.95 | 技术深度 |
| **G2 说得对** | 事实一致率 ≥ 0.92，引用准确率 ≥ 0.95，幻觉拦截率 ≥ 0.90 | 安全可靠 |
| **G3 说得懂** | 可读性指标 FKGL ≤ 10（高中及以下），PEMAT 理解度 ≥ 80% | 用户体验 |
| **G4 看得见** | 每条结论 100% 绑定溯源链接，支持一键查看原文 | 可验证性 |
| **G5 跑得稳** | p95 响应时延 ≤ 3 秒，系统可用性 ≥ 99.5% | 工程质量 |
| **G6 敢上线** | 高风险问题 100% 触发分流路由，合规审计日志 100% 覆盖 | 合规安全 |

### 4.2 北极星指标

> **患者有效理解率** = 患者阅读后能正确复述核心结论的比例（目标 ≥ 80%）

辅助指标：
- 检索→阅读完成率（用户从看到结果到读完核心解释的转化率）
- 溯源点击率（用户查看原文证据的比例）
- 高风险拦截准确率（应拦截的问题中被正确拦截的比例）
- 人工审核升级率（被路由到人工审核的问题比例，目标 5%~15%）

### 5.2 P0 功能详述

#### F-01：自然语言查询入口

| 属性 | 说明 |
|---|---|
| **用户故事** | 作为患者，我希望能用我自己的话提问，而不需要学习医学术语 |
| **输入** | 自然语言文本（中文为主，支持中英混杂） |
| **输出** | 结构化查询意图 |
| **验收标准** | 1) 支持口语化、模糊、含错别字的输入；2) 查询长度上限 2000 字符；3) 响应时间 < 500ms（意图识别阶段） |
| **示例** | 输入："我爸查出肺上有个结节，医生说让随访，这到底严不严重啊？" → 意图：{疾病: "肺结节", 关注点: "恶性风险与随访策略", 风险等级: "中"} |

#### F-02：查询重写与意图识别

| 属性 | 说明 |
|---|---|
| **用户故事** | 作为系统，我需要把患者的口语化问题重写成医学检索友好的结构化查询 |
| **核心逻辑** | 1) QuickUMLS/MetaMapLite 提取候选医学实体；2) Levenshtein 距离模糊匹配 UMLS 词典（阈值 δ）；3) sentence-transformers 计算查询向量与候选实体向量的余弦相似度，确定最匹配的 UMLS CUI |
| **输出** | {原始查询, 重写查询, 医学实体列表[CUI], 检索过滤器{日期范围, 文献类型, 证据等级}, 风险等级} |
| **验收标准** | 实体重召率 ≥ 0.88；高风险意图识别准确率 ≥ 0.95 |

#### F-03：多源文献异构检索

| 属性 | 说明 |
|---|---|
| **用户故事** | 作为患者，我希望系统搜到的是全球最权威的医学文献，而不是随机网页 |
| **数据源** | 见第七章数据源策略 |
| **检索管线** | BM25 倒排索引召回 Top-50 → MedCPT 双编码器语义重排 → 互惠排名融合（RRF）计算最终排名 |
| **性能指标** | 准确率 0.90，召回率 0.90，响应延迟约 1.91 秒（BM25 检索仅 82ms） |
| **验收标准** | 1) 单次检索覆盖 ≥ 3 个权威数据源；2) Top-10 结果中系统综述/指南/RCT 占比 ≥ 60%；3) 支持细粒度过滤（发表日期、文献类型） |

#### F-04：UMLS 语义对齐模块

| 属性 | 说明 |
|---|---|
| **目标** | 消除患者口语化表达与专业文献之间的"词汇孤岛" |
| **方法** | 1) QuickUMLS 轻量级医学实体提取；2) Levenshtein 距离模糊匹配（超过阈值 θ 的词典条目保留为候选概念）；3) 专用文本嵌入模型计算查询向量与候选实体向量的余弦相似度，确定最匹配 CUI |
| **公式** | 相似度 = cos(q⃗, c⃗) = (q⃗·c⃗) / (‖q⃗‖·‖c⃗‖) |
| **验收标准** | 口语化医学术语到 UMLS CUI 的映射准确率 ≥ 0.85 |

#### F-05：结果筛选与去噪

| 属性 | 说明 |
|---|---|
| **去重策略** | 强主键（DOI/PMID/PMCID/NCT/指南编号）+ 弱去重（标题指纹、作者指纹、时间窗口、语义相似度） |
| **来源类型标注** | 患者材料 / 指南 / 系统综述 / RCT / 观察性研究 / 病例报告 / 社论评论 / 试验注册 / 药监信息 / 健康辟谣 |
| **排序策略** | 优先级：患者材料 > 指南 > 系统综述 > RCT > 观察性研究 > 其他；同时考虑时效性与证据等级 |
| **验收标准** | 1) 重复文献去重率 ≥ 95%；2) 过旧文献（>10 年，除非为里程碑研究）自动降权；3) 撤稿文献标记并降权 |

#### F-06：证据分级输出

| 属性 | 说明 |
|---|---|
| **分级标准** | 采用 GRADE 风格四级确定性：高 / 中 / 低 / 极低 |
| **降级因素** | 偏倚风险、不一致性、间接性、不精确性、发表偏倚 |
| **偏倚检测** | 规则先行 + 模型补充：系统综述参照 PRISMA 2020；RCT 参照 RoB 2 域式判断；系统综述方法学质量参照 AMSTAR 2 |
| **输出格式** | 证据等级标签 + 降级原因（面向患者简化表述） |
| **验收标准** | 证据等级自动标注准确率 ≥ 0.80（与人工标注一致） |

#### F-07：多智能体通俗化翻译引擎（核心创新）

| 属性 | 说明 |
|---|---|
| **用户故事** | 作为患者，我希望系统把"5 年 OS 率为 45.2%（95% CI: 38.1-52.3, p<0.001）"翻译成"接受这种治疗的患者中，大约一半的人在 5 年后仍然存活" |
| **架构** | 基于"心智社会"（Society of Mind）理念构建的五位一体 Simplification Loop |
| **收敛控制** | 每轮迭代计算归一化编辑距离 EditDistance(T_i, T_{i-1})，当 < 阈值 ε 时判定收敛退出 |
| **可读性检验** | 集成 ARI、FKGL、SMOG 三大指标，若不合格自动重置参数重新润色，目标 FKGL ≤ 10 |

              └────────────────────────┘

  - readability_check: ARI, FKGL, SMOG
  - fact_consistency_check: 事实与文献金标准对齐
  - citation_binding: 每个关键句绑定 PMID/DOI
output_format: three_layer_structure
  # 一句话结论 + 证据卡片 + 患者通俗解释

## 七、数据源策略

### 7.1 数据源优先级矩阵

数据源策略的第一原则：**权威等级优先、患者适配优先、更新可控优先、许可清晰优先**。

| 优先级 | 数据源 | 主要用途 | 更新策略 | 许可说明 |
|---|---|---|---|---|
| **A** | PubMed / PMC | 文献发现、PMID 主键、开放获取全文 | 每日增量 | E-utilities 公开接口 |
| **A** | Europe PMC | 欧洲文献检索 + 自动实体标注 | 每日增量 | RESTful Web Service |
| **A** | Semantic Scholar | 学术图谱 + 影响力评分 + AI TL;DR | 每日增量 | Graph API |
| **A** | Cochrane PLS | 系统综述通俗化摘要（患者友好） | 每周巡检 + 事件触发 | 订阅产品，优先展示不默认训练 |
| **A** | WHO / NICE 指南 | 临床/公卫 recommendations | 每周巡检 + 版本监测 | 保留版本号 |
| **A** | MedlinePlus | 患者教育、easy-to-read、多语言 | 每日/隔日同步 | Web service 日更 |
| **A** | 国家卫健委 / 中国疾控 | 中文政策、诊疗规范、健康科普、辟谣 | 每周巡检 + 热点触发 | 本地适用性强 |
| **A** | 国家药监局 | 药品/器械监管信息 | 每日/每周巡检 | 监管信息核验 |
| **A** | ChiCTR / ClinicalTrials.gov | 临床试验检索入口 | 每日增量 | 只做检索入口 |
| **B** | 国家癌症中心等专病指南 | 专病权威中文资源 | 月度巡检 | 肿瘤等专病价值高 |
| **C** | 学协会指南、期刊共识 | 补充覆盖 | 月度或人工选编 | 需单独分级标注 |

### 7.2 数据质量控制

| 质控层 | 策略 | 说明 |
|---|---|---|
| **去重** | 强主键（DOI/PMID/PMCID/NCT/指南编号）+ 弱去重（标题指纹/作者指纹/时间窗口/语义相似度） | 将"预印本—期刊版—新闻解读—转载摘要"串到同一证据簇 |
| **来源标注** | 10 类标签：患者材料/指南/系统综述/RCT/观察性研究/病例报告/社论评论/试验注册/药监信息/健康辟谣 | 前台排序优先级据此决定 |
| **证据分级** | GRADE 四级：高/中/低/极低确定性 + 降级原因 | 规则先行 + 模型补充 + 人工抽检 |
| **偏倚检测** | 系统综述参照 PRISMA 2020；RCT 参照 RoB 2；综述方法学参照 AMSTAR 2 | 后台质量门槛，不在前台展示复杂方法学术语 |
| **事实核查** | 四步法：证据锁定生成 → 结论-引用对齐 → 跨源交叉核验 → 中文谣言核验 | 高风险公共卫生议题必查辟谣平台 |

## 八、医疗幻觉防御体系

### 8.1 幻觉分类

| 幻觉类型 | 定义 | 危害 |
|---|---|---|
| **一致性幻觉（Consistency）** | 模型提供的事实与临床证据相违背 | 误诊 |
| **完整性幻觉（Completeness）** | 模型遗漏关键诊断指征或引入无关信息 | 漏诊 |
| **非支持断言** | 标准 RAG 中因信息冗余/时序混乱/概念交叉导致不支持声称 | 误导性结论 |

### 8.2 四层防御管线

```

## 九、安全与合规

### 9.1 合规定位

| 维度 | 定位 |
|---|---|
| **产品分类** | Clinical-Ops AI（临床运营与科普教育级别 AI） |
| **非 SaMD 依据** | 输出完全锚定已公开发表的公认临床指南/文献，不包含 AI 自发的个性化诊断推理 |
| **免除条款** | 临床决策支持系统（CDSS）只要输出完全锚定在 established 临床共识指南中，即可豁免于 SaMD 注册流程 |
| **前台声明** | "本系统提供健康信息与循证文献解释，不提供个体化诊断，不替代医生，不用于急症处置，不自动生成处方" |

### 9.2 隐私安全设计

| 安全层 | 措施 | 标准/依据 |
|---|---|---|
| **前置去标识化** | Safe Harbor De-identification 管线，深度学习 NER 分类器剥离 18 类个人身份敏感标识符 | HIPAA Safe Harbor |
| **生克隔离设计** | 用户身份数据库与医疗文献检索内容数据库物理分库隔离，禁止交叉索引，切断 PHI 合成路径 | PHI 组合产生原则 |
| **零留存协议** | 外部 LLM API 强制配置 Zero-Retention + Data Exclusion 规则，30 天内不可逆物理删除 | HIPAA / PIPL |
| **端到端加密** | 数据传输和持久化存储全面应用 FIPS 140-3 验证的最高级别加密 | FIPS 140-3 |
| **防篡改审计** | tamper-evident 操作级审计日志（智能体标识/操作行为/文献编号/专家签名/毫秒时间戳），强加密存储 ≥ 6 年 | 审计合规要求 |
| **最小化采集** | 默认不留存病历原文；可撤回、可导出、可删除；未成年人单独流程 | PIPL / 未保条例 |

### 9.3 中国合规要点

| 法规 | 核心约束 | 系统应对 |
|---|---|---|
| **《互联网诊疗监管细则》** | 医师本人实名认证接诊；其他人员或 AI 不得冒用、替代医师；严禁 AI 自动生成处方 | 系统不进入互联网诊疗场景；不提供个体化诊断或处方 |
| **《个人信息保护法》** | 医疗健康属于敏感个人信息；需单独同意；需 PIPIA 评估 | 最小化采集 + 单独同意 + PIPIA 前置 |
| **《数据安全法》** | 覆盖数据全流程 | 数据分类分级 + 全链路加密 |
| **《未成年人网络保护条例》** | 不满 14 周岁需监护人同意 + 专门规则 | 未成年人单独流程设计 |

### 9.4 伦理三原则

1. **不要把不确定性藏起来** —— 明确标注证据等级与局限性
2. **不要把群体证据冒充个体建议** —— 默认不做个体诊疗
3. **不要为了用户体验牺牲风险提示** —— 高风险问题必须降级或拒绝

### 10.1 LLM 策略：先指令工程，后轻量微调，最后知识微调

| 阶段 | 策略 | 适用时机 |
|---|---|---|

### 10.2 端侧蒸馏方案（P2）

采用 Patient Knowledge Distillation (PKD-skip) 框架：

- 学生模型拟合教师模型特定中间隐藏层（第 2/4/6 层）的非线性语义特征表征
- 联合优化目标 = 交叉熵损失 + 软目标 KL 散度损失 + 隐藏层激活状态 MSE
- 实测效果：轻量化模型 NER 与概念标准化 F1 逼近教师模型，运行成本降低 101 倍，推理速度提升 12 倍
